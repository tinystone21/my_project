Установка
sudo apt-get install git

Проверка
git --version

Настройка
1. Зарегистрироваться в Github/Gitlab
git config --global user.email "email"
git config --global user.name "username"
2. Проверяем введенные настройки
git config --global user.email
git config --global user.name

Создание ssh-ключа
ssh-keygen:
1. Выбрать файл для ключа("по умолчанию" - Enter)
2. Если ключ уже существует, то выбрать презаписать ключ или нет
3. Ввести и подтвердить пароль от Github и подтвердить его

Копировать публичный ключ
cat ~/.ssh/id_rsa.pub   (по умолчанию)
github.com/settings/profile
-> SSH and GPG Keys
-> Add new SSH key
Вводим имя компьютера и ключ

Инициализируем git
git init

Создаем новый репозиторий в Github
Скопировать ссылку SSH
git remote add origin ссылка

Список привязанных, удаленных репозиториев
git remote -v

Создаем файл .gitignore
Указываем файлы и папки, которые не должны попасть в github
gitignore.io -> python -> копируем содержимое
nano .gitignore -> Вставляем и сохраняем

Команды:
git status - отслеживание изменений.
Отмечаются любые изменения: добавление файлов, изменение содержимого файла, удаление.
Файлы, которые будут добавлены к текущей версии, будут отмечены зеленым.
Остальное будет помечено красным
Изменения в игнорируемых файлах и папках не проверяются

git add - добавление изменений к коммиту (в версию)
git add . - добавление всех файлов
git add file1 file2 - добавление конкретных файлов

git reset - откат изменений
git reset - откат всех изменений из текущей версии
git reset file1 - откать указанных файлов
